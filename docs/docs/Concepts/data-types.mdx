In Langflow, data types define the structure and flow of information between components.

Data types are represented by [port colors](/concepts-components#port-colors), which can be connected to circles of the same colors to exchange data of that type.

## Data

The `Data` data type represents structured data with a main text field and additional metadata. The main text field is indicated by a `text_key`.

Use `Data` when to pass key-value pairs between components.

A `Data` port in a component is represented by a red circle: <Icon name="Circle" size="16" aria-label="Circular component port icon" style={{ color: '#dc2626', fill: '#dc2626' }} />.

`Data` can be serialized _to_ JSON, or created _from_ JSON or other dictionary data, but it's a structured object with validation and methods, not just a plain dictionary.

- **Schema:** [`data.py`](https://github.com/langflow-ai/langflow/blob/main/src/backend/base/langflow/schema/data.py)

**Data attributes:**
- `data`: A Dictionary that stores key-value pairs.
- `text_key`: Which key in `data` is considered the main text.
- `default_value`: Fallback if `text_key` is missing. The default `text_key` is `"text"`.

### Data structure

A Data object stores key-value pairs within the `.data` attribute, where each key is a field name and its value can be any supported data type. `text_key` tells Langflow which key in the data dictionary should be considered the “main text” for that object.

```python
data_obj = Data(
    text_key="text",                    # Field 1
    data={                              # Field 2 (the actual dict)
        "text": "Hello world",
        "name": "Charlie",
        "age": 28
    },
    default_value=""                    # Field 3
)
```

When serialized into JSON, the object appears as below:

```json
{
  "text_key": "text",
  "data": {
    "text": "User Profile",
    "name": "Charlie Lastname",
    "age": 28,
    "email": "charlie.lastname@example.com"
  },
  "default_value": ""
}
```

## DataFrame

The `DataFrame` data type is a pandas [DataFrame](https://pandas.pydata.org/docs/reference/api/pandas.DataFrame.html), which is similar to the tabular data of a CSV file.

Use `DataFrame` to work with data containing multiple rows or records.

A `DataFrame` port in a component is represented by a pink circle: <Icon name="Circle" size="16" aria-label="Circular component port icon" style={{ color: '#ec4899', fill:'#ec4899' }} />.

- **Schema:** [`dataframe.py`](https://github.com/langflow-ai/langflow/blob/main/src/backend/base/langflow/schema/dataframe.py)

**DataFrame attributes**
- **Full pandas compatibility:** All pandas DataFrame methods and functionality work as expected.
- **Langflow integration:** Accepts lists of `Data` objects, dictionaries, or existing DataFrames.
- **Convenience methods:** `to_data_list()`, `add_row()`, `add_rows()`, `to_lc_documents()`, `to_data()`, `to_message()`.
- **Text key support:** Maintains `text_key` and `default_value` attributes for Data object compatibility.

### DataFrame structure

A DataFrame has a tabular data structure with rows and columns.
Keys are columns, and each dictionary in the list is a row.

```json
[
  {
    "name": "Charlie Lastname",
    "age": 28,
    "email": "charlie.lastname@example.com"
  },
  {
    "name": "Alexandra Example",
    "age": 34,
    "email": "alexandra@example.com"
  }
]
```

When represented as tabular data, the preceeding DataFrame is structured like this, with the keys `name`, `age`, and `email`.

| name | age | email |
|------|-----|-------|
| Charlie Lastname | 28 | charlie.lastname@example.com |
| Alexandra Example | 34 | alexandra@example.com |

## Message

The `Message` data type extends `Data` with fields and methods for chat and conversational flows.

The `Message` port in a component is represented by a blue circle: <Icon name="Circle" size="16" aria-label="Circular component port icon" style={{ color: '#4f46e5', fill: '#4f46e5' }} />

Use the `Message` data type to work with chat, conversational flows, or LLM (Large Language Model) input/output, because it provides a consistent structure for chat interactions.

- **Schema:** [`message.py`](https://github.com/langflow-ai/langflow/blob/main/src/backend/base/langflow/schema/message.py)

**Message attributes**
- `text`: Main message content.
- `sender`: "User" or "AI".
- `sender_name`: Display name.
- `session_id`: Chat session identifier.
- `timestamp`: UTC timestamp.
- `files`: List of file paths or images.
- `content_blocks`: Rich content (text, media, code, etc.).
- `category`: "message", "error", "warning", or "info".

### Message structure

A conversational message appears in Langflow as a string, but it is also a structured object that wraps the message content with other metadata.

For example, Langflow displays the string:

```
"Name: Charlie Lastname, Age: 28, Email: charlie.lastname@example.com"
```

The Langflow `Message` data type is actually a dictionary, like the example below.
The `text` field contains the string, while the other fields like `session_id` and `timestamp` contain metadata.

    ```json
    {
      "text": "Name: Charlie Lastname, Age: 28, Email: charlie.lastname@example.com",
      "sender": "User",
      "sender_name": "Charlie Lastname",
      "session_id": "some-session-id",
      "timestamp": "2024-06-01T12:00:00Z",
      "files": [],
      "content_blocks": [],
      "category": "message"
    }
    ```

## View data types in flows

To observe how the different data types are presented in Langflow, use a **Type Convert** component.

1. Connect a **Chat Input** component to a **Type Convert** component.
    This converts the **Message** input from the **Chat Input** component into a Message, Data, or DataFrame type.
2. To run the **Type Convert** component, click <Icon name="Play" aria-hidden="true"/> **Run component**.
3. To inspect the **Type Convert** component's output, click <Icon name="TextSearch" aria-hidden="True" /> **Inspect output**.
