In Langflow, data types are [Pydantic](https://docs.pydantic.dev/latest/api/base_model/) models that define the structure and flow of information between components.

The most common types are **Data**, **Message**, and **DataFrame**.

## Data

The `Data` data type is a Python dictionary of fields. It uses a `text_key` to indicate the main text content.

A `Data` port in a component are represented by a red circle: <Icon name="Circle" size="16" aria-label="Circular component port icon" style={{ color: '#dc2626', fill: '#dc2626' }} />.

`Data` can be serialized _to_ JSON, or created _from_ JSON or other dictionary data.

- **Schema:** [`data.py`](https://github.com/langflow-ai/langflow/blob/main/src/backend/base/langflow/schema/data.py)

**Main attributes:**
- `data`: A Dictionary to store all fields and values.
- `text_key`: Which key in `data` is considered the main text.
- `default_value`: Fallback if `text_key` is missing. The default `text_key` is `"text"`.

In the below example, `text` is the `text_key`, so `"User Profile"` is the main text.

```json
{
  "text": "User Profile",
  "data": {
    "name": "Charlie Lastname",
    "age": 28,
    "email": "charlie.lastname@example.com"
  }
}
```

## DataFrame

The `DataFrame` class is a custom extension of the Pandas [DataFrame](https://pandas.pydata.org/docs/reference/api/pandas.DataFrame.html) class, specifically designed to work seamlessly with Langflow's `Data` objects. The class includes methods for converting between `DataFrame` and lists of `Data` objects.

- **Schema:** [`dataframe.py`](https://github.com/langflow-ai/langflow/blob/main/src/backend/base/langflow/schema/dataframe.py)

**Key features:**
- Accepts lists of `Data`, dictionaries, or existing DataFrames.
- Methods to convert to/from lists of `Data`.
- Integrates with pandas for advanced data manipulation.

### DataFrame structure
A tabular data structure with rows and columns. Keys are columns, and each dictionary in the list is a row.
```json
[
  {
    "name": "Charlie Lastname",
    "age": 28,
    "email": "charlie.lastname@example.com"
  },
  {
    "name": "Alexandra Example",
    "age": 34,
    "email": "alexandra@example.com"
  }
]
```

## Message

The `Message` object extends `Data` with fields and methods for chat and conversational flows. It is the standard type for chat history, LLM responses, and user input.

- **Schema:** [`message.py`](https://github.com/langflow-ai/langflow/blob/main/src/backend/base/langflow/schema/message.py)

**Main attributes (in addition to those from Data):**
- `text`: Main message content.
- `sender`: "User" or "AI".
- `sender_name`: Display name.
- `session_id`: Chat session identifier.
- `timestamp`: UTC timestamp.
- `files`: List of file paths or images.
- `content_blocks`: Rich content (text, media, code, etc.).
- `category`: "message", "error", "warning", or "info".

### Message structure example

A string, such as:
```
"Name: Charlie Lastname, Age: 28, Email: charlie.lastname@example.com"
```
